关于docker for mac 占用大量硬盘的问题

2020-04-13
tools

文章目录

【注意】最后更新于 April 13, 2020，文中内容可能已过时，请谨慎使用。

有一天我更新 xcode 的时候, 系统提示磁盘空间不足, 我打开一看发现 其他 选项占了竟然有150G! 后来扫描了一遍, 发现了吃硬盘的大户: docker for mac 和微信

在docker for mac 上, 安装时的默认选项是占用磁盘60G, 其实并不是弹性分配的, 而是直接先创建一个60G的docker.raw文件占上, 之后再在文件里面做磁盘分配 如果磁盘不够用了, 那么就先拿容器开刀呗, 反正要用的时候可以再创建(不会真有人把需要持久化的数据扔在容器里面了吧?)

  1. 运行清理命令
1
2
3

$ docker system prune -a
中间略过一坨坨的缓存镜像
Total reclaimed space: 10.15GB
  1. 打开docker的系统偏好设置, 把disk拉到最小, 等待重启

至于微信的事情…习惯了, 微信不管是在mac还是在iphone,都是多吃多占的那一个, 看情况删就行
